# Held — UGC Slideshow Direction (2026-07-10)

Held = quiet Christian prayer-partner app. One partner, small daily rhythm, no groups, no guilt.
Status: **prelaunch** — CTA is the **waitlist** (no TikTok account yet; cron built but DISABLED).

## Founder direction (this is the brand's lane)
- **Human-centric UGC**, not the pixel-only Cherry look. **Slide 1 = a real face** reacting to the
  scene (shock / caught-out / quietly emotional). The face is the scroll-stopper.
- **Longer hooks** — 1–2 native sentences, not a punchy tagline.
- **Story / open-loop** is the favored archetype.
- Must sound like a **native TikTok, never an ad.**

## Voice rules (native, not scripted)
- First person, mid-thought openers: "ok so," "not me," "can we talk about," "POV:"
- Exactly **one concrete specific** per hook (the 11pm text, the guy at church, "like 50 people").
- **Zero product mention** until the loop pays off. Held only appears on the final slide.
- Slightly messy grammar on purpose. No clean marketing cadence. No em-dash-heavy ad polish.
- Never trip the forbidden claims: no "God will answer," no guaranteed growth, no "replaces church/counseling."

## Slide structure (6-slide UGC open loop)
1. **Reaction-face HOOK** — real face + longer confessional hook overlaid. Opens the loop.
2. **Setup detail** — concrete, specific ("it was from three weeks ago, she was going through something real").
3. **The turn** — "and then my brain did the thing it always does."
4. **The realization** — the reframe lands ("it wasn't a discipline problem, it was an alone problem").
5. **The quiet answer** — praying alone was the problem → one partner.
6. **Soft reveal + waitlist CTA** — native, e.g. "that's the whole reason I'm on the Held waitlist."

## Reaction-face sourcing (avoid the stocky/AI look)
Ranked best → fastest:
1. **Real creator footage** (best): a person films a genuine reaction to the caption. Most native, highest trust.
   Source via a micro-UGC creator once the account/budget exists.
2. **One reference photo → expression variants** (founder's chosen workflow): keep ONE reference photo of a
   person and reuse that identity across every slideshow with different poses/expressions. Generated by
   `scripts/gen_face_variants.py` (identity-preserving image edit — needs an image API key with reference/edit
   support, e.g. gpt-image-1 edit or FAL InstantID). AVOID: studio lighting, symmetrical framing, model good
   looks, perfect skin — those read as stock/ad and kill trust.
3. **Licensed UGC still** — acceptable fallback.

**NOTHING in a Held slideshow is pixelated** (founder direction, 2026-07-10). Held is fully crisp/photographic
end to end — the pixel-art lane is Cherry-only. Slide 1 = the reaction face; story-beat slides (2–6) are crisp,
quiet photographic scenes (warm cream / soft blue / candle gold / muted violet mood, symbolic not literal, no
faces, no clipart, no literal Jesus, no guilt imagery). Pool lives in `backgrounds/held_pool/`.

## Face library (how the renderer picks a face)
Drop expression photos of the person into `faces/held/`, named by mood:
`shock.png`, `caught_out.png`, `emotional.png`, `dread.png`, `recognition.png`, `late_night.png`, `neutral.png`.
`render_slides.py` reads each post's `reaction_face` and auto-picks the closest mood file (falls back to
`neutral.png`, then the first file). Override per-render with `--face <file-or-dir>`.

## How it's generated
`src/slideshow_engine/held_hooks.py` holds a curated bank of 12 full open-loop SEQUENCES (hand-written native
voice — NOT slot-filled, because slot-filling reads robotic). `generate(n)` rotates the bank; `--append` adds
the next N unused. Output → `campaigns/held/latest_campaign.json`. Each post carries `reaction_face` (the slide-1
expression) and `format: ugc_reaction`.

  → all slides crisp, slide 1 = face from `faces/held/`). Flags: `--face`, `--face-slides`, `--crisp-slides`
  (`auto`/`all`/`none`/list). Cherry stays pixelated. Verified end-to-end on post 0.

  `faces/held/` manually.

  Held integration ID once it exists.
